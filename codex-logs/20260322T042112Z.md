## Topic
- Refactor shape-assembly integration into a clear host/robot split while preserving the real-robot navigation pipeline.

## Decisions
- Added `formation_msgs` as the shared protocol package with `ShapeTask` and `RobotFormationStatus`.
- Added `formation_host` for host-side strategic publication of shared formation tasks from `/gather_center`.
- Added `formation_robot` for robot-side local staging-goal bridging into each robot's own `move_base`.
- Kept `fm2_gather` as the host strategic center selector, but added `publish_goal` launch control and strategic-mode replan behavior so host no longer needs to push per-robot goals.
- Updated `shape_assembly_swarm.py` to consume `ShapeTask`, switch shape metadata at runtime, and publish local robot formation status.
- Converted `shape_assembly_host.launch` and `shape_assembly_agent.launch` into compatibility wrappers over the new host/robot packages.
- Updated `motion_navigate_multi4.launch` to pass `agent_id` into the robot formation wrapper and to forward host shape parameters.

- Host-side shape heading is currently parameter-driven in `formation_host`; automatic heading search on the host has not been added yet.
